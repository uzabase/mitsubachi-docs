# バナー

ユーザーに情報・成功・警告・エラーなどの通知を確実に届けるためのコンポーネント。

- Do / Don't は Figma ページ内の「仕様」フレームを参照

## 使い分け

Snackbar より情報密度が高く、ユーザーが内容を読み取るまで表示し続ける通知に使う。
一瞬で消えるフィードバックには Snackbar を使う。

| status | 使いどころ |
|--------|-----------|
| **Information** | 補足説明・お知らせなど、知っておくと便利な情報 |
| **success** | 操作の成功や完了の通知 |
| **warning** | 注意が必要な状態の警告 |
| **error** | エラーや問題の通知 |

### 使用を避けるケース
- 削除などの重大なアクションの確認で、ユーザーの決断が必須となるケース（→ Dialog を使う）
- 瞬間的な操作フィードバック（→ Snackbar を使う）
- 表示領域を圧迫するほど長文になるケース（→ 別ページやヘルプで説明する）

## 振る舞い
- Banner は自動で消えない。ユーザーが閉じるボタンを押したタイミングで非表示になる
- メッセージ内にテキストリンクを挿入し、補足情報や関連ドキュメントへ誘導できる
- 必要に応じてアクションボタンを配置し、別のステップへ誘導できる

## アクセシビリティ
- 動的に表示するバナーには `role="alert"` または `aria-live` を付ける
- 閉じるボタン（×）には `aria-label="閉じる"` を付ける

## 並べ方
- コンテンツエリアの上部中央に配置する（ヘッダーやナビゲーションとは重ならない位置）
- 複数表示する場合は縦に 8px 間隔で積む
