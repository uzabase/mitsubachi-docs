# スナックバー

一時的なフィードバックメッセージを画面端に表示するコンポーネント。

## Figma
- コンポーネント: https://www.figma.com/design/kHQNLM1dnk0EhZwOKBEBkL/Base-Component-Speeda-3.1-MITSUBACHI?node-id=5607-6067
- Do / Don't は Figma ページ内の「仕様」フレームを参照

## 使い分け

### サイズ
| size | 用途 |
|------|------|
| **small** | 短いメッセージ（1行程度） |
| **medium** | やや長いメッセージ（2行まで） |

### 表示位置
| viewport | 位置 |
|----------|------|
| **desktop** | 画面右上 |
| **phone** | 画面下部中央 |

## 振る舞い
- 自動的に消える（auto-dismiss）。表示時間は 3〜5 秒程度
- 複数のスナックバーは縦に積み重なる
- テキストは最大 2 行まで。それ以上は省略または別のコンポーネントを使う
- 閉じるボタン（×）で手動で消すこともできる
- アクションボタン（例：「元に戻す」）を1つだけ配置できる

### 使用シーン
- 操作の完了通知（「保存しました」「コピーしました」）
- 軽微なエラー（「接続に失敗しました。再試行してください」）
- 情報の通知（「新しい通知があります」）

### 使用を避けるケース
- 重要な確認が必要なメッセージ（→ dialog を使う）
- 長文の説明が必要な場合（→ inline-notification を使う）
- ユーザーの操作をブロックする必要がある場合（→ dialog を使う）

## アクセシビリティ
- `role="status"` または `role="alert"` を使用する
- 重要度が高い場合は `aria-live="assertive"`、低い場合は `aria-live="polite"`
- 閉じるボタンには `aria-label="閉じる"` を付ける
- 自動で消える前に読み上げが完了するよう、表示時間を十分に取る

## 並べ方
- 画面端から 16〜24px のマージンを取る
- 複数表示時は 8px の間隔で積み重ねる
- 他のUIを遮らない位置に表示する
