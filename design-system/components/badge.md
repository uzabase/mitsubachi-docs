# バッジ

未読数や通知の有無を示す小さなインジケーター。

- Do / Don't は Figma ページ内の「仕様」フレームを参照

## 使い分け

アイコンボタンやメニュー項目に重ねて、未読・更新・通知があることを示す。

| パターン | 使いどころ |
|---------|-----------|
| **ドットのみ** | 数を伝える必要がない場合。「何か新しいものがある」ことだけを示す |
| **1桁（1-9）** | 未読数や通知数を具体的に示す |
| **2桁（10-99）** | 同上 |
| **3桁以上（100+）** | 100以上は「99+」のように上限表示にする |

## 配置ルール

組み合わせるコンポーネント（アイコンボタン等）を基準として、**右上**に少し重なるように配置する。

- 数字を表示するタイプは桁数によって幅が変わるため、基準位置を決めておく
- ドットのみの場合は小さいため、アイコンに近づけすぎない

## アクセシビリティ
- バッジの数値はスクリーンリーダーで読み上げられるよう `aria-label` で補足する
  - 例: `aria-label="3件の未読通知"`
- 色だけで情報を伝えない。数字やドットの形状で判別できるようにする

## 並べ方
- 親要素（アイコンボタン等）の右上角を基準に配置
- 複数のバッジを同じ要素に付けない（1つの要素に1つのバッジ）
