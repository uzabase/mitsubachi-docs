# インライン通知

ユーザーの操作に対するエラーや注意事項を、画面の文脈の中で即座に表示するコンポーネント。

## Figma
- コンポーネント: https://www.figma.com/design/kHQNLM1dnk0EhZwOKBEBkL/Base-Component-Speeda-3.1-MITSUBACHI?node-id=5638-4139
- Do / Don't は Figma ページ内の「仕様」フレームを参照

## 使い分け

### Banner との違い
| コンポーネント | 用途 |
|--------------|------|
| **Inline Notification** | 特定の入力や操作に紐づくフィードバック。フォーム内のエラーなど |
| **Banner** | ページ全体に関わる通知。システムメンテナンスのお知らせなど |

### バリアント
| variant | 見た目 | 使いどころ |
|---------|-------|-----------|
| **primary** | アイコン + 背景色あり | 重要度が高いメッセージ |
| **secondary** | アイコンのみ、背景なし | 補足的なメッセージ |

### ステータス
| status | 使いどころ |
|--------|-----------|
| **information** | 補足説明や案内 |
| **success** | 操作が成功したことを伝える |
| **warning** | 注意が必要な状態 |
| **error** | 入力エラーや問題の通知 |

## 振る舞い
- 自動で消えない（ユーザーに内容を読んでもらう必要があるため）
- 閉じるボタンは基本的に付けない（誤って閉じて重要情報を見落とさないため）
- 親要素の幅に従って表示される

### 使用を避けるケース
- 複数の選択肢を提示したい場合（→ Banner を使う）
- 重要な行動を促したい場合（→ Banner や Dialog を使う）
- 画面全体に影響する重大な通知（→ Banner を使う）

## アクセシビリティ
- エラーメッセージには `role="alert"` を付ける
- 関連する入力フィールドと `aria-describedby` で紐付ける
- 色だけでなくアイコンとテキストで状態を伝える

## 並べ方
- 関連する入力フィールドや操作対象の近くに配置する
- フォームの場合は、エラーのあるフィールドの直下または上部に表示
