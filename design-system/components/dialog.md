# ダイアログ

ユーザーの注意を引き、確認や入力を求めるモーダルウィンドウ。

## Figma
- コンポーネント: https://www.figma.com/design/kHQNLM1dnk0EhZwOKBEBkL/Base-Component-Speeda-3.1-MITSUBACHI?node-id=4179-586
- Do / Don't は Figma ページ内の「仕様」フレームを参照

## コンポーネント一覧

| コンポーネント | 役割 | いつ使うか |
|---------------|------|-----------|
| **action-dialog** | 確認や選択を求める | 削除確認など、ユーザーの決断が必要な場合 |
| **information-dialog** | 情報を伝える | 閉じるボタンのみ。通知や説明を表示する場合 |
| **form-dialog** | フォーム入力を含む | 保存・キャンセルボタンを配置。入力を求める場合 |

## 使い分け

### デバイス別表示
| viewport | full-screen | 説明 |
|----------|:-----------:|------|
| **desktop** | false | 画面中央にオーバーレイ表示 |
| **phone** | false | 画面中央にオーバーレイ表示（幅が狭い） |
| **phone** | true | 画面全体を覆う（フルスクリーン） |

## 振る舞い
- 背景はオーバーレイで暗くする
- ダイアログ外のクリックでは閉じない（明示的な操作が必要）
- Escape キーで閉じられるようにする（form で入力中の場合は確認を挟む）

## アクセシビリティ
- `role="dialog"` と `aria-modal="true"` を付ける
- `aria-labelledby` でタイトルと関連付ける
- 開いたときにフォーカスをダイアログ内に移す
- 閉じたときにフォーカスを元の要素に戻す
- Tab キーでダイアログ内をループする（背景要素にフォーカスが移らない）

## 並べ方
- タイトルは上部に配置
- 本文・フォームは中央に配置
- ボタンは下部（フッター）に配置
- 主要アクションのボタンは右側、キャンセルは左側
