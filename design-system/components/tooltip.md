# ツールチップ

UI 要素の補足情報をホバー時に一時表示するコンポーネント。

## Figma
- コンポーネント: https://www.figma.com/design/kHQNLM1dnk0EhZwOKBEBkL/Base-Component-Speeda-3.1-MITSUBACHI?node-id=6656-2364
- Do / Don't は Figma ページ内の「仕様」フレームを参照

## 使い分け

アイコンボタンなど、見た目だけでは意味が伝わりにくい要素に付ける。
操作に必須の情報はツールチップに頼らず、常に見えるラベルで表示する。

## 幅について
- max-width を設定し、長すぎるツールチップを防ぐ
- 短いテキストは1行で表示
- 長いテキストは max-width で折り返す

## z-index
- すべての要素の中で一番上に表示する
- 他の要素と重なることはない

## トリガーからの距離
- トリガーとなるコンポーネント（例: icon-button など）のターゲットエリアから **8px** とする
- 指やカーソルと重ならない（hidden タイプ）カーソルエリアも同様

## インタラクション
- 表示までの時間: とか
- 消えるまで: 0.1s で消える
- アニメーション: ease-out

## 配置
| 配置 | 説明 |
|------|------|
| **Left** | トリガーの左側に表示 |
| **Center** | トリガーの中央に表示 |
| **Right** | トリガーの右側に表示 |
| **下に表示** | トリガーの下に配置 |
| **上に表示** | トリガーの上に配置 |

- 画面端では自動的に配置を調整する（画面スピードを上げる）

## Do / Don't

### Do
- 近接する他のコンポーネントが隠れない場所に配す

### Don't
- 近接する他のコンポーネントが隠れる場所は避ける

## UIライティング
- どこかのタイミングで他かを一緒に考える

## アクセシビリティ
- トリガー要素に `aria-describedby` でツールチップと紐付ける
- キーボードフォーカスでも表示されるようにする（ホバーだけに頼らない）
- Escape キーで閉じられるようにする

## 並べ方
- トリガー要素の近くに表示する（重ならないよう配置を自動調整）
- 他の UI を大きく遮らない位置にする
