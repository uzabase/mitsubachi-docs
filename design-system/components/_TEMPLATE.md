# コンポーネントドキュメント テンプレート

> 新しいコンポーネントのドキュメントを書くときは、このテンプレートをコピーしてください。
>
> **書く前に確認すること:**
> - Figma のコンポーネントフレームを見て、variant / size / state / selected などのプロパティを洗い出す
> - **同一ページ内に複数のコンポーネントがあるか確認する**（例: `link-tag` と `read-only-tag`）
> - Figma ページ内の「仕様」フレームで Do/Don't を確認する（仕様フレームがないコンポーネントもある。その場合は Do/Don't もドキュメントに書く）
> - Storybook も確認し、使い分けやアクセシビリティのヒントを探す
>
> **書き方の方針:**
> - Figma MCP で読み取れるものはドキュメントに書かない:
>   - バリアント・サイズ・色・寸法・影・角丸などの視覚情報 → コンポーネントフレームから取得可能
>   - Do / Don't → 「仕様」フレームから取得可能
> - Figma MCP では読み取れないものだけをドキュメントで補う:
>   - **コンポーネント一覧** — 同一ページ内に複数コンポーネントがある場合、それぞれの役割
>   - **使い分け** — variant は見えるが「いつどれを選ぶか」の判断基準
>   - **選択状態の対応表** — どの variant で selected が使えるかのルール
>   - **アクセシビリティ** — aria 属性、キーボード操作
>   - **並べ方** — 複数コンポーネント間の配置順序・間隔
> - 該当しないセクションは削除してよい
> - `{...}` の部分を実際の内容に置き換える
> - **`_` で始まるコンポーネントは Figma 内部用なので、ドキュメントには含めない**
