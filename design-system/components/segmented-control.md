# セグメンテッドコントロール

複数の選択肢から1つを選ぶ切り替え型コンポーネント。

## Figma
- コンポーネント: https://www.figma.com/design/kHQNLM1dnk0EhZwOKBEBkL/Base-Component-Speeda-3.1-MITSUBACHI?node-id=7684-7456
- Do / Don't は Figma ページ内の「仕様」フレームを参照

## 使い分け

### バリアント
| variant | 用途 |
|---------|------|
| **text** | テキストラベルで選択肢を表示 |
| **icon** | アイコンのみで選択肢を表示 |

### 選択肢の数
| item | 説明 |
|------|------|
| **2〜5** | 2個から5個までの選択肢に対応 |

### デバイス別表示
| viewport | 説明 |
|----------|------|
| **desktop** | 標準サイズ（高さ 32px） |
| **phone** | やや大きめ（高さ 36px） |

## 振る舞い
- 必ず1つのセグメントが選択状態になる（single-select のみ）
- 別のセグメントを選択すると、元の選択は自動的に解除される
- 複数選択は許容しない（複数選択が必要な場合は filter-chip の multi-select や checkbox を使う）

### 使用を避けるケース
- 選択肢が6つ以上ある場合（→ Tab や Select-box を検討）
- ラベルが長すぎて省略が必要な場合（→ Tab や Select-box を検討）

## アクセシビリティ
- `role="tablist"` と `role="tab"` を使用する
- 選択中のセグメントには `aria-selected="true"` を付ける
- 矢印キーでセグメント間を移動できるようにする
- icon バリアントの場合は `aria-label` でラベルを付ける

## 並べ方
- 各セグメントの幅は均一にする（バラバラにしない）
- セグメントのラベルは省略しない
- 最大幅は親コンポーネントの幅とする（横スクロールが発生しないように）
- 最大幅でもラベルが収まらない場合は折り返しを許容
