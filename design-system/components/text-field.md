# テキストフィールド

1行のテキスト入力を受け付けるフォームコンポーネント。

## Figma
- コンポーネント: https://www.figma.com/design/kHQNLM1dnk0EhZwOKBEBkL/Base-Component-Speeda-3.1-MITSUBACHI?node-id=3-325
- Do / Don't は Figma ページ内の「仕様」フレームを参照

## 使い分け

### バリアント
| バリアント | 用途 |
|------------|------|
| **default** | 標準的なテキスト入力 |
| **with-icon** | 左側にアイコン付き（検索など） |
| **with-button** | 右側にボタン付き（クリア、送信など） |
| **with-unit** | 右側に単位表示付き（円、%など） |

### text-area との違い
| 項目 | text-field | text-area |
|------|------------|-----------|
| 行数 | 1行 | 複数行 |
| 用途 | 名前、メールなど短いテキスト | コメント、説明文など長いテキスト |

## 状態
| state | 説明 |
|-------|------|
| **default** | 通常状態 |
| **hover** | マウスオーバー時 |
| **focus** | フォーカス時（青い枠線） |
| **error** | エラー時（赤い枠線） |
| **disabled** | 操作不可（グレーアウト） |

## 振る舞い
- フォーカス時に枠線の色が変わる
- エラー時は赤い枠線とエラーメッセージを表示
- 入力内容が長い場合は水平スクロール（折り返しなし）

## アクセシビリティ
- `<label>` 要素と関連付ける
- エラー時は `aria-invalid="true"` と `aria-describedby` でエラーメッセージを関連付ける
- プレースホルダーをラベルの代わりにしない
- 必須項目には `aria-required="true"` を付ける

## 並べ方
- ラベルはテキストフィールドの上に配置
- エラーメッセージはテキストフィールドの下に配置
- 関連するフィールドはグループ化する（16px 間隔）
