# 設計原則

mitsubachi-ui で画面を作るときに守る、4 つの基本原則。

---

## 1. 一貫性（Consistency）

同じ意味を持つ要素には、同じスタイルを適用する。

- **主要アクション** → 常にほぼ黒の塗りつぶし + 白テキスト（neutral-button primary）
- **破壊的アクション** → 常に赤の塗りつぶし + 白テキスト（danger-button primary）
- **リンク** → 常に青テキスト（`#2666bf`）
- **補足テキスト** → 常にグレー + 小さめのフォントサイズ
- **無効状態** → 常に薄いグレー背景 + 薄い文字色

ある画面で「削除」が赤いボタンなら、別の画面でも「削除」は赤いボタンにする。

---

## 2. 階層（Hierarchy）

視覚的な重みで情報の優先度を表現する。

| 要素 | 重みの付け方 |
|------|-------------|
| 見出し | 太字 + ほぼ黒（`rgba(0,0,0,0.84)`） |
| 本文 | 通常ウェイト + ほぼ黒 |
| 補足・キャプション | 通常ウェイト + グレー + 小さいフォントサイズ |
| 主要ボタン | 黒い塗りつぶし（最も目を引く） |
| 補助ボタン | 枠線のみ、または背景なし（控えめ） |

1 画面に primary ボタンは 1 つ。複数あると優先度が曖昧になる。

---

## 3. 余白（Spacing）

余白はグルーピングを表現する手段。関連する要素は近く、無関係な要素は遠く。

4px 刻みのスケールを使う。

| 値 | 主な用途 |
|-----|---------|
| 4px | アイコンとテキストの間 |
| 8px | インライン要素間、ボタン同士の間隔 |
| 12–16px | フォーム要素間、コンパクトなパディング |
| 24px | カード内のパディング |
| 32–48px | セクション間 |
| 64px 以上 | ページセクション間 |

迷ったら「近すぎないか？遠すぎないか？」で判断する。同じグループ内の余白は統一する。

---

## 4. アクセシビリティ

- **コントラスト** — テキストと背景のコントラスト比は WCAG AA（4.5:1）以上を維持する
- **フォーカスリング** — インタラクティブ要素には必ずフォーカスリングを付ける（黒い二重リング: `0 0 0 2px #fff, 0 0 0 4px #191919`）
- **タップターゲット** — ボタン・リンクの最小タッチ領域は 44×44px
- **色だけに頼らない** — エラーは赤 + テキスト（またはアイコン）で伝える。色覚多様性への配慮
- **ラベル** — アイコンだけのボタンには `aria-label` を必ず付ける
- **プレースホルダー** — 入力のヒントには使ってよいが、ラベルの代わりにしない
