# カラー運用ルール

> 具体的な色の値や CSS 変数名は `tokens/tokens.json` および [mitsubachi-tokens.css](https://github.com/uzabase/mitsubachi-token) を参照。
> ここでは **使い分けの判断基準** を定義する。

---

## カラーの階層

mitsubachi-ui のカラーは 2 階層で構成されている。

| 階層 | 役割 | 使い方 |
|------|------|--------|
| **Primitive** | ベースパレット（neutral, blue, tomato, red, green, yellow） | 直接使わない。semantic の裏側にある素材 |
| **Semantic** | 用途に紐づいた色（text, object, surface, border 等） | **画面で使うのはこちら** |

画面を作るときは常に semantic トークンで考える。「この青は `blue-100` だから…」ではなく「これはリンクだから `text-link`」と判断する。

---

## テキスト色の使い分け

| 場面 | 意味 | 目安の見た目 |
|------|------|-------------|
| 見出し・本文 | 通常のテキスト | ほぼ黒（`rgba(0,0,0,0.84)`） |
| 補足・ラベル | やや弱いテキスト | 濃いグレー（`rgba(0,0,0,0.68)`） |
| キャプション・注釈 | さらに弱いテキスト | グレー（`rgba(0,0,0,0.54)`） |
| プレースホルダー | 未入力の手がかり | 薄いグレー |
| 無効状態 | 操作できないことを示す | かなり薄いグレー（`rgba(0,0,0,0.35)`） |
| 白抜き | 暗い背景上のテキスト | 白 |
| リンク | クリック可能なテキスト | 青（`#2666bf`） |
| エラー・必須 | 問題や必須マーク | 赤（`#c92812`） |

---

## 背景色（surface）の使い分け

| 場面 | 目安の見た目 |
|------|-------------|
| ページ背景 | 白（`#ffffff`） |
| 全体の下地・座布団 | 薄いグレー（`#f8f8f8`） |
| ホバー | ごくわずかに暗くなる |
| 無効状態 | ほぼ透明なグレー |
| ボタン primary | ほぼ黒 |
| ボタン danger | 赤 |
| エラー領域 | 淡い赤（`#ffedeb`） |
| 成功領域 | 淡い緑（`#dff5ea`） |
| 警告領域 | 淡い黄（`#fcf6d4`） |
| 情報領域 | 淡い青（`#e8f2ff`） |

---

## ボーダー色の使い分け

| 場面 | 目安の見た目 |
|------|-------------|
| 強いボーダー（ヘッダー下など） | ほぼ黒 |
| 通常のボーダー（カード、区切り線） | 薄いグレー（`rgba(0,0,0,0.13)`） |
| 無効状態 | さらに薄いグレー |
| エラー | 赤 |
| チェック済み（ラジオ、チェックボックス） | 青 |

---

## セマンティックカラーの使用条件

セマンティックカラー（成功・警告・エラー・情報）は **状態を伝える場合のみ** 使う。装飾目的では使わない。

### ✅ Do

- フォームバリデーションのエラーメッセージに赤を使う
- 保存成功のトーストに緑を使う
- 色とテキスト（またはアイコン）を **併用** して状態を伝える

### ❌ Don't

- 見た目のアクセントとしてエラーの赤を使う
- ブランドカラーの代替として成功の緑を使う
- **色だけ** で状態を伝える（色覚多様性への配慮）
